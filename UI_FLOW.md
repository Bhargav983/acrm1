Design the UI for an application called:

Astrology Research ERP

UI inspiration should come primarily from ERPNext.

Do not design it like a horoscope website.

Do not use astrology-themed graphics, zodiac wheels, planets, stars, mystical backgrounds, dark themes, or decorative spiritual elements.

The application should feel like a professional ERP system used by astrologers and researchers.

Use:

* ERPNext-style layout
* Bootstrap 5 style components
* Light theme
* Sidebar navigation
* List views
* Detail pages
* Timeline activity feeds
* Dashboard cards
* Professional business application design

Main Navigation:

* Dashboard
* Customers
* Astrology Data
* Knowledge Base
* Verifications
* Remedies
* Consultations
* Reports
* Administration

Dashboard:

Show cards:

* Total Customers
* Total Knowledge Records
* Total Verifications
* Active Remedies
* Today's Follow Ups

Show recent activities.

Show recent consultations.

Show recent verification activities.

Customer Module:

Customer List page should be the primary entry point.

Columns:

* Name
* Mobile
* Nakshatra
* Pada
* Occupation
* Education
* Last Consultation

Actions:

* View
* Edit

Customer Profile Page:

Use ERPNext style tabs.

Tabs:

1. Basic Details
2. Birth Details
3. Astrology Data
4. Knowledge Tracking
5. Remedies
6. Consultation Notes
7. Timeline

Timeline should show all customer activity chronologically.

Astrology Data Screen:

Display:

* Lagna
* Moon Sign
* Sun Sign
* Nakshatra
* Pada
* Mahadasha
* Antardasha

Display source:

* Manual
* API
* Import

Knowledge Base Module:

Display records in ERPNext-style list view.

Support:

* Predictions
* Remedies
* KP Rules
* BNN Rules
* Past Life Observations
* Research Questions
* Case Studies

Knowledge Detail Screen:

Show:

* Category
* Section

* Expected Result

Customer
→ Matching Knowledge Records

Allow researcher notes.

Assign remedies.

Track:

* Start Date
* Follow Up Date

* Outcome

ERP-style consultation records.

Show:

* Consultation Date

Reports Module:

* Customer Report

Administration:

* Roles
* Users

Design Principles:

* ERPNext inspired
* Fast data entry
* Research focused
* Professional
* Clean tables
* Strong filtering
* Minimal clicks
* Mobile responsive
* Easy for researchers to use all day

The application should feel like an ERP system for astrology research rather than a traditional astrology application.
